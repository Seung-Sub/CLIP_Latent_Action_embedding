## 요약: 이 연구는 무엇이고, 어디에 서 있는가

당신의 아이디어는 이렇다. CLIP이 이미지와 언어를 한 공간에 정렬했듯, **7-DoF × 16 timestep의 action chunk를 인코더에 통과시켜 CLIP image latent(768차원 — 언급하신 728은 ViT-L/14 기준 768의 오기로 보임)와 같은 공간으로 보내되, 그 목표를 "행동 전후 이미지 latent의 차이" `Enc(a) ≈ z_{t+k} − z_t`로 삼는다.** action 자체는 이미지와 정렬될 수 없지만, action이 세계에 일으킨 *변화*는 이미지 latent의 변위로 나타나므로, 그 변위를 의미 공간의 좌표로 쓰겠다는 발상이다.

130편을 검증 조사한 결론을 세 문장으로 압축하면: **(1)** 이렇게 raw action chunk를 frozen 사전학습 vision-language 공간의 변위 벡터에 회귀시키는 논문은 2026년 7월 기준 발견되지 않았다 — 신규성이 성립한다. **(2)** 다만 그 구조의 모든 부품은 개별적으로 검증되어 있다 — action 임베딩이 latent state 변화를 예측하게 하는 것(DynE), frozen 시각 latent 차이가 action으로 예측 가능하다는 것(DINO-WM, V-JEPA 2-AC), 새 모달리티를 frozen CLIP 공간에 바인딩하는 것(MotionCLIP, ImageBind, MotionBind). **(3)** 가장 큰 리스크는 CLIP이라는 공간 자체다 — 2024–25 문헌은 dynamics 용도로 CLIP을 일관되게 회피하며(MAE/DINOv2/SigLIP-2 선호), 이 선택을 "언어와의 사전 정렬을 물려받기 위한 의도적 선택 + DINOv2 대조 실험"으로 정면 돌파해야 한다.

# 1부. 문헌 지형: action을 latent로 보내온 네 갈래 길

설계를 논하기 전에, "action을 latent space로 보낸다"는 아이디어가 지금까지 어떤 모습으로 존재해 왔는지 지도를 그려야 한다. 이 지도가 곧 related work의 뼈대이자 신규성 주장의 근거가 된다. 조사 결과 기존 연구는 네 계보로 나뉜다.

## 1.1 계보 A — 시연 데이터의 action/trajectory 압축 (skill latent)

가장 오래된 계보다. 시연에 담긴 action 시퀀스를 저차원 latent로 압축해 재사용한다.

| 논문 | 입력 | 인코더 | latent | 목적 |
|---|---|---|---|---|
| Play-LMP (1903.01973, CoRL 2019) | state+action 윈도우 | bi-RNN CVAE ⚠️세부 | 연속 "plan" | 다봉성 흡수, play 재사용 |
| SPiRL (2010.11944, CoRL 2020) | action만 H=10 ⚠️ | LSTM VAE ⚠️ | 연속 10-d ⚠️ | 하류 RL의 action space + 탐색 prior |
| OPAL (2010.13611, ICLR 2021) | state+action 부분궤적 | bi-GRU CVAE ⚠️ | 연속 8-d ⚠️ | offline RL 분포이탈 방지, 시간 추상화 |
| CompILE (1812.01483, ICML 2019) | 가변 segment | 미분가능 분할+인코딩 | 연속/이산 | 구성적 재사용 |
| ACT (2304.13705, RSS 2023) | action chunk+고유수용 | transformer CVAE | 연속 32-d, 추론 시 z=0 | **순수 다봉성 흡수** (학습 후 폐기) |
| TAP (2208.10291, ICLR 2023) | state 조건 궤적 | transformer VQ-VAE | 이산 VQ | latent 코드 공간에서 planning |
| SkiMo (2207.07560, CoRL 2022) | action 시퀀스+dynamics | VAE+dynamics 모델 | 연속 | skill 공간 모델기반 planning |
| PRISE (2402.10450, ICML 2024) | obs-action 쌍 | VQ(코드 10)+**BPE**(vocab 200) | 이산 2단 | **"시간 추상화=시퀀스 압축"** (원문 확인) |
| QueST (2407.15840, NeurIPS 2024) | **action만** T=32 | 1D causal conv+self-attn+**FSQ** | 이산 8토큰 | 전이 가능한 skill vocabulary |
| VQ-BeT (2403.03181, ICML 2024) | action chunk | MLP+**RVQ 2단** | 이산+연속 offset | 다봉성 포착, 추론 5× 가속 |
| LISA (2203.00054, NeurIPS 2022) | 궤적+언어 | VQ | 이산 skill 코드 | 언어와 상관된 해석가능 skill ⚠️ |
| FAST (2501.09747) / BEAST (2506.06072) | action chunk | **비학습** DCT+BPE / B-spline | 이산 토큰 | LM 호환 토큰화 (학습형 인코더 회피) |

주목할 실증 두 가지. QueST는 **인코더를 action-only로 유지하고 관측은 prior에만** 넣었을 때가 최선이었고, 관측을 디코더에 넣으면 오히려 성능이 저하됨을 원문에서 확인했다. 그리고 이 계보의 latent 형태는 연속 CVAE(2019–21) → 이산 VQ/FSQ/BPE(2022–24, 동기가 "RL prior"에서 "LLM 호환 토큰"으로 이동) → 다시 연속(2025–26: CLAM 2505.04999, villa-X의 codebook center)으로 진자 운동을 해왔다.

## 1.2 계보 B — RL을 위한 action representation

이 계보는 "임베딩이 어떤 구조를 담아야 하는가"라는 질문에 답해 왔고, 그 답이 본 설계의 직계 조상이다.

- **Chandak PG-RA**(1902.00183, ICML 2019): action latent를 **(s, s') 전이 쌍을 설명하는 변수**로 학습 — P(a|s,s')의 생성적 복원(원문 확인). "비슷한 효과의 action은 가깝게"의 원조.
- **HyAR**(2109.05490, ICLR 2022): **state 잔차(s'−s) 예측**을 보조 손실로 latent를 조형(초록 확인).
- **DynE**(1908.09357, ICLR 2020): **action chunk의 임베딩이 k-step 후의 latent state를 예측하도록** 변분 학습(원문 확인) — **본 설계와 구조적으로 가장 가까운 선행**. 단 latent state 공간을 처음부터 학습한다.
- **Koopman 계열**(서베이 2408.04200, RoboKoop 2409.03107): 효과 = latent의 **선형 변위** z'=Az+Bu — "action=latent 변위"의 수학적 극단.
- 그 외: PLAS(2011.07213, 데이터 support 인코딩), LASER(2103.15793), Wolpertinger(1512.07679, 비학습 임베딩+kNN), TD-MPC2(2310.16828, 대조군: action을 임베딩하지 않고 raw로 dynamics에 입력).

## 1.3 계보 C — 비디오 프레임 차이에서 latent action 추출 (LAM)

본 설계의 **거울상**이다. 이들은 프레임 쌍 (o_t, o_{t+k})에서 latent action을 역으로(inverse) 뽑고, forward 모델이 그 latent로 미래 프레임을 복원하게 하는 정보 병목으로 학습한다.

핵심 검증 사실들: Genie(2402.15391)는 VQ codebook |A|=8, 그리고 "decoder는 LAM에 학습 신호를 주기 위해서만 존재하며 추론 시 (VQ codebook 제외) LAM 전체가 폐기된다"(원문). LAPO(2312.10812, ICLR 2024)는 "IDM은 o_t와 o_{t+1}의 **차이만을** z에 인코딩하도록 학습된다"(원문 verbatim)를 명시했고, VQ 병목이 정책에 더 좋은 latent를 만든다는 ablation도 제시했다. Moto(2412.04445, ICCV 2025)는 **frozen ViT의 patch feature 위에서** M-Former(학습형 query 8개)+VQ(vocab 128)로 motion token을 만들었다 — 기질(frozen 사전학습 feature) 선택이 본 설계와 같다. LAPA(2410.11758, ICLR 2025)는 NSVQ로 양자화한 latent action을 VLM이 예측하게 한 뒤 실제 action으로 finetune했다.

2024–26 확장: **IGOR**(2411.00785)는 사람·로봇 공용 latent action(언어 정렬은 policy 경유 후행), **UniVLA**(2505.06111, RSS 2025)는 **언어 지시를 disentangling 신호로** 쓰되 latent는 DINOv2 feature 공간에 구축, **GO-1/ViLLA**(2503.06669)는 VLM→latent action→action expert의 3단 인터링구아, **GR00T N1**(2503.14734)은 사람 비디오를 latent action으로 라벨링해 "LAPA"라는 별도 embodiment로 취급(원문 확인), **AdaWorld**(2503.18938, ICML 2025)는 latent action 조건 world model, **CLAM**(2505.04999)은 연속 latent+action 디코더 공동학습, **DreamGen**(2505.12705)은 LAM을 생성 비디오의 주석 도구로 사용.

경고 논문 두 편이 특히 중요하다. **LAOM**(2502.00379, ICML 2025): distractor가 있으면 LAM latent는 노이즈를 인코딩하며, **2.5%의 action 감독**만 넣어도 4.2× 회복된다. **"What Do LAMs Actually Learn?"**(2506.15691, NeurIPS 2025): 선형 분석으로 **LAM ≈ 프레임 차이의 PCA**이며 외생 변동(조명·카메라)을 흡수함을 보였다. 이 경고는 Δz에도 그대로 적용된다.

## 1.4 계보 D — 사전학습 공간으로의 정렬 (본 설계가 속한 곳)

- **MotionCLIP**(2203.08063, ECCV 2022): 사람 모션을 transformer AE의 [CLS]식 첫 토큰으로 요약해 **frozen CLIP 공간에 cosine 정렬 + 복원 손실**(관절·vertex·속도 L2) — 본 설계의 가장 가까운 방법론적 원형(원문·코드 확인: latent 512, λ_vel=100).
- **GRIF**(2307.00117, CoRL 2023): 언어를 "goal image가 아니라 **start↔goal 이미지의 변화**"에 정렬(초록 verbatim) — 언어↔Δ 정렬의 직접 선행.
- **CLIP-RT**(2411.00508, RSS 2025): action을 **언어 모션 프리미티브**로 표현해 CLIP 공동 공간에 대조 임베딩 — 공간은 맞으나 action 정의가 감독된 언어 라벨.
- **MotionBind**(NeurIPS 2025): 사람 모션 인코더를 기존 사전학습 LanguageBind 공간에 바인딩 — "새 모달리티를 frozen 공간에 넣는" 최신 실증(로봇 아님).
- **LARA**(2606.07100 ⚠️제목·ID만 확인): latent action 표현과 VLA 내부 표현의 정렬 — 착수 전 정독 필수.

## 1.5 지형이 말해주는 것: 공통점, 차이점, 그리고 빈칸

**공통점.** 네 계보를 관통하는 사실: ① 감독은 거의 전부 자기 재구성(AE/VAE/VQ)이다. ② 정보 병목의 강도가 성패를 가른다. ③ **latent 공간은 예외 없이 새로 학습된다** — 기질이 pixels → DINOv2 feature(UniVLA) → promptable VLM feature(VLM-LAM 2601.22714)로 점점 사전학습 쪽으로 이동해 왔지만, 사전학습 vision-language **공동 공간 자체**에 latent가 사는 논문은 없다. ④ state 조건화가 준보편적이다. ⑤ latent의 운명은 폐기(ACT의 z, Genie의 LAM) 아니면 새 action space로 승격, 둘 중 하나다.

**모달리티 확장 시도.** 언어가 latent에 닿는 경로는 넷뿐이고 모두 간접적이다: 조건화 입력(LangLfP 2005.07648, UniVLA), 상관 유도(LISA), **policy 경유 후행 정렬**(IGOR — "align latent actions with natural language *through the foundation policy model*", 초록 verbatim), action을 아예 언어로 정의(CLIP-RT).

**본 설계의 위치.** 따라서 차별점은 세 가지로 정리된다. **(1) target 공간**: frozen CLIP vision-language 공동 공간의 변위 벡터 — 미점유. **(2) 방향**: LAM은 video→latent action(inverse·재구성), skill latent는 action→자기 압축(재구성)인 반면, 본 설계는 **action→시각 의미 효과(forward·회귀)** — 감독 신호가 자기 자신이 아니라 외부 고정 의미 공간이다. **(3) 목적**: RL prior도 압축 토큰도 아닌 **cross-modal 접지 자체**. 이 조합(LAM식 latent + MotionBind식 사전학습 공간 바인딩)은 조사 범위에서 미개척으로 확인됐다.

# 2부. 전제 점검: 이 가설은 성립하는가

지형에서 설계로 내려오기 전에, `Enc(a) ≈ Δz`라는 목표 자체가 딛고 선 가정들을 점검한다.

## 2.1 "Δz에 action 정보가 있다"는 근거

순방향 증거: **DINO-WM**(2411.04983, ICML 2025)은 frozen DINOv2 feature 위에서 `z' = F(z, a)`를 학습해 zero-shot planning에 성공했고, **V-JEPA 2-AC**(2506.09985)는 frozen V-JEPA 2 encoder 위에 7-DoF delta-EE 조건 predictor를 학습해 실제 Franka 조작에 성공했다. 즉 frozen 시각 latent의 시간 변화는 action으로 예측 가능한 구조를 가진다. 역방향 증거는 1.3절의 LAM 계보 전체이고, 의미론적 증거로 **VIP**(2210.00030)·**R3M**(2203.12601)·**TCN**(1704.06888)은 임베딩의 시간차가 진행도·보상 신호가 되도록 학습 가능함을, **LIV**(2306.00958)는 이를 CLIP-style 공간으로 확장 가능함을 보였다.

## 2.2 리스크 ① — CLIP 공간 자체의 적합성 (최대 리스크)

- **Eyes Wide Shut**(2401.06209, CVPR 2024): **CLIP-blind pairs** — 명백히 다른 두 이미지가 CLIP에서 거의 같은 임베딩. 그리퍼 2cm 이동 정도의 변화는 Δz가 0에 가까울 수 있다.
- **The Temporal Trap**(2502.03270): 정적 이미지용 사전학습 표현의 "temporal entanglement" — 정책 성공률은 latent가 task-progression 신호를 담는 정도와 강한 상관.
- **Burns et al.**(2312.12444, CoRL 2024)·**Theia**(2407.20179): 조작에 필요한 것은 CLIP식 의미 불변성이 아니라 공간적으로 정밀한 feature.
- 실제로 미래-latent 정렬 계열(GR-1 2312.13139, Seer 2412.15109, FLARE 2505.15659)은 모두 CLIP이 아닌 MAE ViT/VQGAN/SigLIP-2를 dynamics 공간으로 썼다.

**완화책(제안)**: (a) 16-step chunk의 양 끝 프레임 차가 크도록 시간 간격 설계(Moto도 "significant visual differences"를 위해 프레임 다운샘플, 원문 확인); (b) Δz의 L2 정규화·표준화; (c) **같은 파이프라인을 DINOv2로도 돌리는 비교 ablation을 논문의 핵심 실험으로**; (d) 그래도 CLIP을 쓰는 이유(언어와의 사전 정렬 상속 → 4부)를 명확히 서사화.

## 2.3 리스크 ② — Modality gap

**Mind the Gap**(2203.02053, NeurIPS 2022): CLIP의 image와 text 임베딩은 각각 좁은 cone에 분리되어 있다(cone effect + contrastive 온도). 새 모달리티를 CLIP 공간에 붙이면 그 latent도 자기 cone에 갇히는 경향이 있다(ImageBind에서도 emergent 정렬은 직접 학습보다 약함).

단, 본 설계의 target은 절대 위치가 아니라 **차이 벡터 Δz**다. 같은 모달리티 내부의 차이이므로 image cone의 평균 성분이 상쇄된다 — 이는 **C3**(2401.08567, ICLR 2024)가 gap 해소책으로 처방한 "각 모달리티의 평균 임베딩을 빼라(Collapse)"를 target 정의가 이미 내장한 셈이다. **(해석/제안: 이 연결은 본 보고서의 논증이며, 논문화하려면 Δz 분포와 cone 평균의 관계를 실증할 것.)** 단 gap은 cross-modal *비교* 시점에 다시 등장한다 — Enc(a)→Δz 회귀 학습 자체는 gap의 영향이 없지만, 4부의 text↔Δz 비교가 정확히 gap에 노출되는 지점이다. CLIP 임베딩 산술이 텍스트 의미와 대응하는지 직접 실험한 선행으로 **Embedding Arithmetic**(2112.03162)이 있으니 사전 점검에 활용하라.

## 2.4 리스크 ③ — 정보 비대칭 (pivot bottleneck)

Wav2CLIP(2110.11499)의 교훈: 이미지에 보이는 것만 살아남는다. Δz에는 **결과(outcome)** 정보만 있고 경로·속도·힘 정보는 없다. `Enc(a)`가 Δz *정확히*가 되도록만 학습하면 chunk 내부 정보를 버리는 것이 최적해가 된다. 따라서 **정렬 손실과 복원 손실의 균형이 이 설계의 본질적 긴장**이며, action 복원 디코더(3부 §5.1)가 필수 장치다. MotionCLIP이 정확히 같은 구조(recon + cosine 정렬)로 이 긴장을 다뤘다.

## 2.5 리스크 ④ — 불량설정(ill-posedness): Δz는 state에 의존한다

같은 7-DoF chunk라도 장면·잡은 물체·카메라에 따라 Δz는 달라진다. `Enc(a) ≈ Δz`는 one-to-many 문제이고, state 없이 학습하면 인코더는 기껏해야 E[Δz|a]를 배운다. 인용된 모든 IDM/FDM(LAPO·Genie·Moto·DINO-WM·V-JEPA 2-AC)이 관측을 조건으로 받는 이유다. **처방(제안)**: predictor h에 z_t를 조건으로 주입 — `h(Enc(a), z_t) ≈ Δz`. 이때 1.1절 QueST의 실증(인코더는 action-only, 관측은 prior에만 — 디코더 조건화는 오히려 저하)과 결합하면 결론이 선명해진다: **조건화는 predictor 쪽에, Enc 자체는 action-only로 유지.** 여기에 2506.15691(LAM≈프레임 차이 PCA, 외생 노이즈 흡수)과 LAOM(소량 감독의 효과)의 처방 — 보조 inverse head, 데이터 정제, ≈2.5% 감독 — 을 실험 설계에 반영한다.

# 3부. 설계: 인코더, 손실, 학습 트릭, 그리고 RVQ

## 3.1 공통 전처리

**차원별(per-dimension) 1–99 percentile quantile 정규화**(OpenVLA 2406.09246 원문 확인; FAST 동일). min-max(RT-1)보다 outlier에 강건하다. action은 절대 좌표보다 delta-EE/relative가 표준(LAPA·OpenVLA 계열).

## 3.2 인코더 아키텍처

**제안 A — 주력: Transformer + [CLS] 풀링.**
```
a ∈ R^{16×7} → timestep별 linear proj (7→d, d=256~512)
→ sinusoidal PE → 학습형 [CLS] prepend (17 토큰)
→ Transformer encoder 3~4층 (heads 8, FFN 4d)
→ [CLS] 출력 → 2층 MLP projection → R^768
```
근거가 가장 많다: ACT의 CVAE 인코더가 정확히 이 구조([CLS]+qpos+k action tokens, 4층·hidden 512 — 원문·코드 확인), MotionCLIP도 transformer AE의 첫 토큰을 CLIP 공간에 정렬, TMR(2305.00976)은 학습형 분포 토큰 풀링, ImageBind(2305.05665)의 IMU 브랜치(저차원 시계열→1D conv→transformer ⚠️세부)도 같은 패턴. QueST의 1D causal strided conv 프론트엔드도 대안 부품이다.

**제안 B — 필수 baseline: MLP.** 112차원 flatten → 3층 MLP → 768. VQ-BeT가 action chunk 인코더로 plain MLP로 충분함을 보였다(공식 코드 확인). 길이 16은 transformer의 이점이 안 나올 수 있는 길이라 리뷰어가 반드시 요구할 비교군.

**제안 C — 보조: 주파수 특징 병행(제안).** FAST의 발견 — 로봇 action은 매끄럽고 시간 상관이 높아 DCT 저주파 계수에 정보가 집중 — 을 활용해 차원별 DCT 상위 계수를 병행 토큰/concat 입력으로 추가. 저주파(궤적 전체 형태)가 Δz(결과 변화)와 대응할 것이라는 직관.

## 3.3 정렬 손실

| 선택지 | 근거 | 판정 |
|---|---|---|
| cosine/MSE 회귀 → frozen Δz | CLIP-KD(2307.12732, CVPR 2024): frozen CLIP 증류에서 단순 MSE mimicry가 "surprisingly well"(원문; 단 동일 모달리티 증류 — cross-modal 일반화는 본 보고서의 추론). REPA(2410.06940, ICLR 2025): frozen DINOv2에 cosine. MotionCLIP: cosine. FLARE: `1−cos` (원문 pseudocode) | **주손실** |
| InfoNCE (batch 내 negative) | Wav2CLIP: cross-projection contrastive가 증류 안정화(원문). MDT의 CLA도 InfoNCE | **보조손실** (λ≈0.1부터) |
| CLIP 동결 | LiT(2111.07991, CVPR 2022): "locked image tower가 최고"(verbatim). AudioCLIP(2106.13043): CLIP까지 풀면 성능은 오르나 공유 공간 붕괴 | **동결** |
| collapse 방지 | SimSiam(2011.10566): stop-gradient 필수+predictor(원문). BYOL(2006.07733): EMA target | target이 frozen이라 위험 낮음. **BYOL식 비대칭 predictor** 권장 |

종합: `L_align = 1 − cos(h(Enc(a), z_t), Δz_norm) + λ_nce·InfoNCE` — h는 2층 predictor(§2.5의 state 조건화 지점), Δz는 L2 정규화, CLIP 완전 동결.

## 3.4 학습 트릭: 보조 모듈을 붙였다 버리기

**(1) Action 복원 디코더 (강력 권장).** ACT가 원형: CVAE 인코더로 chunk를 z로 압축하고 **테스트 시 인코더를 버리고 z=0**. MotionCLIP은 recon(관절+vertex+**속도** L2)과 정렬 cosine을 병용 — 속도항이 시간 패턴을 보존한다(원문·코드 확인). 역할은 §2.4의 정보 손실 방지: `Dec(Enc(a)) ≈ a` (L1/SmoothL1)로 chunk 내부 정보를 latent에 강제로 남긴다. CVAE로 만들면 저차원 latent의 posterior collapse(2306.05023)에 주의 — KL/β-annealing 필수, 회귀 목표면 deterministic AE로 충분.

**(2) 미래 latent 디코더 (권장).** 문헌은 두 진영이다. *보조로 쓰고 버리는 진영*: MDT(2407.05996, RSS 2024)의 Masked Generative Foresight — 미래 프레임 masked patch를 복원하는 ViT 디코더를 학습에만 쓰고 "without additional inference costs"(verbatim), 실환경 0.51→0.58; REPA — 정렬 head 폐기, 수렴 17.5× 가속; LAM 계열의 FDM — Genie의 "decoder는 학습 신호용으로만 존재". *생성기가 planner인 진영(비권장)*: SuSIE(2310.10639 — 통용되는 2311.10090은 오류)·UniPi(2302.00111)는 diffusion 생성기를 배포 시에도 실행. 제안: `Enc(a)`와 z_t를 조건으로 z_{t+16}을 예측하는 **latent-level** 소형 디코더(DINO-WM의 "픽셀 재구성 없이"와 FLARE 방향)를 붙이고 배포 시 버린다.

**(3) 정책 통합 시의 트릭.** FLARE: action denoiser의 **중간 layer**(8층 중 6층)에서 future token을 미래 관측 임베딩과 cosine 정렬(λ 가중 보조손실), 최대 26% 개선, 너무 이른 layer는 역효과(원문 확인); target 인코더는 EMA(0.995). Moto·villa-X: action head가 latent에 과의존하지 않도록 **latent로의 attention 50% 확률 마스킹** — villa-X는 "crucial in practice"(원문). C3식 노이즈 주입: downstream 학습 시 임베딩에 가우시안 노이즈를 더해 gap 잔차에 강건하게. **CLASP**(1806.09655, ICLR 2019)의 composability: Δz는 telescoping으로 정확히 가법적(z16−z0 = (z16−z8)+(z8−z0))이므로 `Enc(a_{1:8})+Enc(a_{9:16}) ≈ Enc(a_{1:16})` 정규화가 구조와 일치 — 이 가설을 명시적 손실로 만든 유일한 선행.

배포 시 남는 것은 `Enc` 하나다. action 디코더, 미래 디코더, predictor h, InfoNCE projection — 전부 학습 후 폐기한다.

## 3.5 RVQ를 적용한다면: 세 가지 삽입 위치와 구체 구조

VQ-VAE(1711.00937)의 이산 병목을 직렬 잔차로 확장한 것이 RVQ다(SoundStream 2107.03312): N_q개의 codebook이 각각 이전 단계의 잔차를 양자화해, 코드 스택이 하나의 연속 벡터를 **coarse-to-fine으로 전개**한다. 깊이 D면 유효 codebook이 K^D로 지수적으로 커지고 학습도 안정된다(RQ-VAE 2203.01941, 원문 확인). 문제는 "어디에 꽂느냐"다. 결론부터: **주 목표(연속 Δz 회귀)의 경로에는 넣지 말고(ⓐ 비권장), 보조 head(ⓑ)와 별도 action tokenizer(ⓒ)로만 채택하라.** 이유 — 회귀 target이 외부 고정·연속이므로 경로상 양자화는 손실이 측정하는 바로 그 양에 제거 불가능한 왜곡 하한을 만들며(LAPO의 "VQ 병목 이득"은 latent가 자유로울 때의 이야기라 전이되지 않음), "양자화가 latent action 품질을 저하시킨다"는 직접 반증(2502.00379, 원문 확인)과 "학습형 VQ tokenizer는 tedious to train"(FAST, verbatim)이라는 실무 경고도 있다.

**구조 ⓑ — Δz의 RVQ 토큰화를 보조 분류 head로 (1회 ablation 가치, 학습 후 폐기).**
TIGER(2305.05065, NeurIPS 2023)가 실증한 선례를 그대로 옮긴다: 768차원 frozen 의미 임베딩(Sentence-T5)을 RQ-VAE로 3~4개 "Semantic ID" 토큰으로 만들어 LM이 예측하게 했다.
```
[사전 단계] 학습 데이터의 Δz 코퍼스에 RQ-VAE 학습 (본 학습과 분리, 수 분 소요)
  Δz(768) → RVQ: level-1 codebook(K₁=256) → 잔차 → level-2 codebook(K₂=256)
  → 각 Δz가 (c₁, c₂) 토큰 쌍으로 표현됨. c₁ = "어떤 의미 방향의 변화인가"(coarse),
    c₂ = 세부 보정. 단일 공유 codebook + EMA 갱신 (RQ-VAE 방식).

[본 학습] Enc(a) 뒤에 분류 head 분기:
  L_aux = CE(head₁(Enc(a)), c₁) + β·CE(head₂(Enc(a)), c₂)   (β≈0.5)
  총손실 = L_align(회귀, §3.3) + λ_aux·L_aux
[배포] head₁·head₂와 RQ-VAE 전부 폐기 — Enc만 남음.
```
왜 돕는가: 저데이터에서 순수 회귀는 그래디언트가 밋밋하지만, coarse 분류(c₁)는 "의미 방향" 단위의 뚜렷한 학습 신호를 준다 — REPA·MDT가 보여준 "보조 목표가 주 목표의 수렴을 가속"하는 패턴의 이산 버전이다(이 조합 자체는 제안). 주의: 이것을 **주 target으로 승격하면 안 된다** — 양자화 오차가 action decoder가 읽는 공간에 직접 유입되고, 소규모 로봇 데이터에서 codebook collapse 위험이 있다.

**구조 ⓒ — Action BiCodec: action tokenizer로서의 RVQ + Δz 증류 (언어 결합으로의 확장 경로).**
시연이 다봉(multimodal)이어서 회귀 head가 여러 유효한 chunk의 평균을 내버리는 것이 실패 원인일 때, VQ-BeT식 action tokenizer가 검증된 해법이다(N_q=2, codebook 8~16, EMA, L1 recon, 연속 offset head — 원문 확인: "primary codes는 coarse clustering, secondary는 fine-grained"). 여기에 음성 쪽 두 설계를 이식해 본 설계와 접합한다: Spark-TTS(2503.01710)의 BiCodec은 음성을 **semantic 토큰(내용) + global 토큰(화자 속성)**으로 분해했고, Moshi(2410.00037)의 Mimi 코덱은 **RVQ 1단계에 semantic 정보를 증류**하고 나머지 레벨이 acoustic 세부를 담게 했다(원문 확인).
```
a ∈ R^{16×7} → 1D causal conv 인코더 (QueST식) → 연속 latent u
→ RVQ level-1: 양자화 출력 q₁에 증류 손실 L_distill = 1−cos(proj(q₁), Δz_norm)
   ⇒ level-1 토큰 = "이 chunk의 의미적 효과" (semantic 스트림, CLIP 공간과 접지됨)
→ RVQ level-2(+): 잔차 양자화
   ⇒ 속도·리듬·컴플라이언스 등 Δz에 안 보이는 정보 (style 스트림, §2.4의 정보 비대칭 수용)
→ 디코더: 코드 합 → action chunk 복원 (L1) + 연속 offset head (VQ-BeT식)
```
이 구조의 가치: ① 다봉성은 이산 코드가 흡수하고, ② semantic/style 분해가 §2.4의 긴장(정렬 vs 복원)을 **레벨 분리로 구조화**하며, ③ level-1 토큰이 CLIP 공간에 접지되어 있으므로 4부의 언어 결합(text↔토큰 검색, VLA의 이산 vocabulary)으로 바로 이어진다. 로봇 action tokenizer에 semantic/style 분해는 아직 없다(FAST는 단일 스트림; OmniSAT 2510.09667은 부분공간별 RVQ지만 의미 접지가 없음). 단 이는 Δz 회귀 서사를 강화하기보다 **경쟁하는 별도 기여**로 프레이밍해야 한다(제안).

**공통 안정화 도구.** 채택 시: k-means 초기화 + EMA + dead-code reset(SoundStream/EnCodec/VQ-BeT 계열), NSVQ(LAPA 채택), rotation trick(2410.06424, ICLR 2025), SimVQ(2411.02038, ICML 2025 — collapse 원인을 "disjoint codebook optimization"으로 규명). 가장 간단한 대안은 **FSQ**(2309.15505, ICLR 2024): codebook 학습·보조손실 없이 차원별 반올림 — "codebook collapse가 없고 commitment loss·reseeding 불필요"(verbatim). QueST가 실제로 FSQ를 채택했다. 저GPU에서 양자화가 필요하면 **FSQ 또는 2-level RVQ+EMA**.

**RVQ 토큰을 LM에 넣을 때(구조 ⓒ의 후속).** 다층 코드를 시퀀스로 펼치면 길이가 L배가 된다 — VALL-E(2301.02111)의 해법은 **"layer 임베딩을 합산하라, 펼치지 말라"**(NAR 입력에서 각 레벨의 임베딩 테이블 출력을 합산해 길이 유지), RQ-Transformer는 **spatial + depth transformer** 분리로 비용을 T+T·D로 억제한다. VLA에 이식할 때 그대로 쓸 수 있는 스케줄이다.

# 4부. Language와 LLM: 오디오와 음성-LLM의 교훈으로 설계하기

이제 언어다. 질문은 둘이었다 — action latent를 언어와도 정렬할 수 있는가, 그리고 정렬이 어렵다면 이미 잘 학습된 LLM의 문맥 이해를 어떻게 빌려올 것인가. 답은 오디오 도메인이 먼저 걸어간 길에 있다.

## 4.1 오디오가 알려주는 세 가지 경로

| 경로 | 오디오 선례 | 조건 | 로봇 대응 |
|---|---|---|---|
| ① 새 인코더 → frozen CLIP 공간 (image 피벗) | Wav2CLIP(2110.11499): frozen CLIP+오디오 인코더, cross-projection contrastive, text 정렬 상속("makes zero-shot audio classification possible"). SpeechCLIP(2210.00705): frozen HuBERT+CLIP 사이 소형 브릿지. ImageBind(2305.05665): "only image-paired data is sufficient to bind" — audio↔text **emergent** 정렬 | 언어 라벨 불필요, 데이터 효율 최고(Wav2CLIP ~10% 데이터) | **본 설계의 경로.** 단 오디오는 절대 임베딩 정렬이고 본 설계 target은 Δz — CLIP text encoder는 Δz에 대해 학습된 적 없으므로 text↔Δz emergent 정렬은 **검증 필요한 가설**(GRIF도 언어↔Δ를 별도 학습으로 달성) |
| ② 언어와 직접 joint contrastive | CLAP(2206.04769)·LAION-CLAP(2211.06687): 13만~63만 쌍 필요 | 대량 언어 라벨 | 한 랩 데이터로 비현실적. LAION-CLAP의 keyword-to-caption(메타데이터→자동 캡션)만 이식 |
| ③ 이산 토큰 → LM | VALL-E: 코덱 토큰을 LM으로("conditional language modeling rather than continuous signal regression") | LM 학습 비용 | RT-2(2307.15818)·OpenVLA·LAPA가 이 경로. §3.5 구조 ⓒ의 level-1 토큰이 접점 |

## 4.2 음성-LLM 수업이 주는 구조적 동형 지도

음성인식·합성 교안(L10~12)의 설계 공간은 VLA와 구조적으로 동형이다. "audio"의 자리에 "action"을 넣으면 그대로 성립한다.

| 음성의 축 | 음성 대표작 | VLA 대응물 (검증) |
|---|---|---|
| 입력: 코덱 토큰 vs 인코더 임베딩+어댑터 — "코덱은 압축 목표지 의미 목표가 아니다", 이해 태스크는 임베딩 승리 | SALMONN(2310.13289, 원문 확인: frozen Whisper+BEATs → window Q-Former → frozen LLM+LoRA, 대조 정렬 단계 없음), Qwen-Audio(2311.07919) | VLA 입력측은 이미 어댑터 방식. **action을 '이해 입력'으로 넣는 chunk encoder+어댑터는 미개척** — 본 설계의 Enc가 바로 의미-목표(meaning-target) action 인코더 |
| 출력 A: LLM이 토큰 직접 방출 (vocab 확장; sequential/interleaved/parallel 스케줄) | Spark-TTS, CosyVoice 2(2412.10117, 5:15 interleaving 원문 확인), Mini-Omni(2408.16725, text-delay parallel 원문 확인) | RT-2/OpenVLA/FAST. interleaved=EO-1(2508.21112), parallel=MM-ACT(2512.00975) |
| 출력 B: hidden state → 함께 학습한 디코더 (vocab 불변) | LLaMA-Omni(2409.06666, ICLR 2025): hidden→upsample→**NAR CTC** 디코더, 4 GPU·3일 미만(원문 확인) | π0(2410.24164)의 action expert, CogACT(2411.19650), GR00T N1 |
| "입력측은 싸고 출력측은 무겁다" | 교안 L11 결론 | **Knowledge Insulation**(2505.23705): action expert를 naive하게 붙이면 VLM 지식 손상 → 백본은 이산 FAST 토큰으로 co-train, expert gradient는 **stop-gradient로 절연**(원문 확인) |
| cascade의 한계: 지연·파라언어 손실·오류 전파 | Moshi(2410.00037) 초록: "text being the intermediate modality… non-linguistic information such as emotion is lost" | Hi Robot(2502.19417, ICML 2025)=로봇판 cascade. Steerable Policies(2602.13193): 언어 인터페이스가 "fundamentally limits how much VLM reasoning can steer low-level behavior"(verbatim). π0.5(2504.16054)는 hierarchy를 같은 백본에 내재화 |
| Inner Monologue: 시간 정렬 텍스트를 오디오 토큰의 prefix로 → 품질 대폭 향상 | Moshi (원문 확인) | RT-H(2403.01823): "language motion"을 action 앞에 예측. ECoT(2407.08693): 추론 후 행동 **+28%p**(원문 확인) |
| full-duplex 이중 스트림 | Moshi | **대응물 없음 — 완전 미개척** |

## 4.3 이 지도에서 나오는 설계 (우선순위 순)

**아이디어 1 — Action BiCodec (§3.5 구조 ⓒ와 동일 실체).** semantic 토큰의 정의를 CLIP Δz 정렬로 삼는 것이 음성의 BiCodec/Mimi 증류와 본 설계를 접합하는 지점이다. semantic 토큰이 CLIP 공간에 접지되므로 text와의 비교·검색이 즉시 가능하다. 기존 아이디어를 버리지 않고 언어 결합으로 확장하는 경로.

**아이디어 2 — 시간 정렬 inner monologue for actions (미개척 확인).** RT-H와 ECoT는 "언어 먼저, 행동 다음"의 이득을 실증했지만 둘 다 시간 정렬 스트림이 아니다(에피소드/쿼리 단위, ECoT는 지연이 큼). Moshi처럼 **action chunk마다 그 chunk에 시간 정렬된 짧은 언어 토큰을 몇 step 앞서 같은 스트림에서 방출**하는 VLA는 발견되지 않았다. 구현 재료는 검증돼 있다: Mini-Omni의 text-delay parallel decoding + Real-Time Chunking(2506.07339, NeurIPS 2025: 현재 chunk 실행 중 다음 chunk 생성, freeze+inpaint). 라벨은 RT-H식 자동 생성(궤적 통계→motion 문구).

**아이디어 3 — 정렬이 안 될 때의 우회로: LLM hidden-state conditioning.** 오디오의 결정적 선례는 SALMONN이다 — 음성-텍스트 대조 정렬 단계가 아예 없고, Q-Former 요약을 frozen LLM(+LoRA)에 주입해 의미 이해를 전부 LLM에 위임한다. 로봇에서 같은 패턴이 2024–26 주류(π0·CogACT·GR00T N1·RoboFlamingo 2311.01378 — 단 RoboFlamingo의 ablation상 VLM을 완전히 얼리면 저하되므로 LoRA 경로는 열어둘 것, 원문 확인). 통합 시 Knowledge Insulation 레시피(stop-gradient 절연)를 그대로 적용하고, 저GPU 대안으로 LLaMA-Omni의 NAR+CTC 디코더의 action판(hidden→upsample→NAR chunk 디코더)은 로봇 대응물이 없어 그 자체로 논문거리다.

**아이디어 4 — 논문 서사: "action의 파라언어 손실".** Moshi(음성: 텍스트 중간 표현은 감정을 잃는다) → Steerable Policies(로봇: 언어 인터페이스가 low-level 조향을 근본 제한) → **힘·강성·속도 프로파일은 텍스트 병목을 통과하지 못한다**(이를 명시 분석한 논문은 미발견 — 인용 가능한 빈칸). 본 설계의 Δz-latent 인터페이스가 정확히 이 병목의 대안이라는 인용 사슬이 서론의 동기부여가 된다.

**실행 순서 (저GPU).** ① Δz 정렬 인코더(3부) 완성 + action↔text emergent 검색을 **명시적으로** 측정(ImageBind 교훈: emergent는 직접 학습보다 약하므로 정량화 필수). ② GRIF식 instruction↔Δz InfoNCE를 데이터 있는 만큼 추가(라벨 부족 시 keyword-to-caption). ③ 정렬 품질이 목표 미달이면 SALMONN/π0 패턴으로 전환 — Enc(a)를 Q-Former로 요약해 frozen LLM+LoRA에 soft prompt 주입.

# 5부. 종합: 파이프라인, 실험 계획, 유효성

## 5.1 추천 파이프라인

```
[학습]
a ∈ R^{16×7} ─ quantile norm ─→ Enc (Transformer+[CLS] / MLP baseline)
                                    │
              ┌─────────────────────┼──────────────────┬─────────────────┐
              ▼                     ▼                  ▼                 ▼
      predictor h(·, z_t)    action decoder     future-latent decoder   RVQ 보조 head (ⓑ, 선택)
              │               (L1 recon, 버림)   (z_{t+16} 예측, 버림)    (CE on (c₁,c₂), 버림)
              ▼
   L_align = 1−cos(h(Enc(a), z_t), Δz_norm) + λ_nce·InfoNCE
   L_comp  = ‖Enc(a₁)+Enc(a₂) − Enc(a₁⊕a₂)‖        (CLASP식, 선택)
   (+ L_text = GRIF식 instruction↔Δz InfoNCE — 데이터 있는 만큼)

[배포] Enc만 유지 → VLA action 표현 / 보상·검색 / LLM soft-prompt
[확장] 구조 ⓒ Action BiCodec (semantic=Δz 증류 / style=잔차) → 언어 결합·VLA 토큰화
CLIP: 완전 동결 (LiT) · Δz: L2 정규화 · k: Δz norm이 충분히 크도록
```

## 5.2 우선순위 ablation (리뷰 대비)

1. **CLIP vs DINOv2 target 공간** (§2.2 — 논문의 승부처)
2. Transformer vs MLP 인코더 (VQ-BeT 근거로 필수 요구될 것)
3. cosine 회귀 vs InfoNCE vs 병용 (CLIP-KD vs Wav2CLIP 상반 증거의 실증)
4. 보조 모듈 각각의 기여: action recon / future decoder / RVQ head ⓑ (MDT식 ablation)
5. state 조건화 위치: h에 z_t 주입 vs 없음 (§2.5 ill-posedness 검증)
6. Δz 정규화 방식, chunk 길이 k (Δz SNR과의 관계)
7. action↔text emergent 검색 정량 평가 (§4.3 실행 순서 ①)

## 5.3 2026년 유효성과 경쟁 감시

"작은 모달리티 인코더 + frozen 강자 타워 + [CLS] 풀링 + cosine 정렬" 패턴은 REPA(ICLR 2025)·FLARE(2025)·V-JEPA 2(2025)까지 유효하다. 연속 latent 회귀는 추세와 부합한다(CLAM·villa-X·UniSkill). CLIP을 dynamics 공간으로 쓰는 것만이 문헌의 흐름을 거스르는 선택이며, 이것이 리스크이자 기여 지점이다. **착수 전 반드시 정독할 경쟁 논문 3편**: JALA(2602.21736, CVPR 2026 — VLA 문맥과 IDM latent 정렬), VLM-LAM(2601.22714 — promptable VLM feature 위 LAM), LARA(2606.07100 ⚠️제목만 확인). "사전학습 표현 × latent action"의 교차점이 빠르게 좁혀지고 있으므로 신속한 실험 착수를 권한다.
